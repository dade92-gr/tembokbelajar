A. Review Arsitektur Von Neumann

Arsitektur Von Neumann adalah desain dasar komputer modern yang diperkenalkan oleh John von Neumann pada tahun 1945. Arsitektur ini terdiri dari empat komponen utama yang saling terhubung.

John von Neumann: "Komputer adalah mesin yang memproses data berdasarkan instruksi yang tersimpan dalam memori."

Alur Data dalam Arsitektur Von Neumann

INPUT Keyboard, Mouse, Scanner
CPU Prosesor (Otak Komputer)
OUTPUT Monitor, Printer, Speaker
MEMORY (RAM) Penyimpanan sementara
STORAGE HDD/SSD

Komponen dan Fungsinya

📌 Komponen Utama Arsitektur Von Neumann:
• CPU (Central Processing Unit): Memproses data dan menjalankan instruksi. Terdiri dari ALU, CU, dan Register.
• Memory (RAM): Menyimpan data dan program sementara saat komputer menyala.
• Input Device: Perangkat untuk memasukkan data ke komputer (keyboard, mouse, scanner).
• Output Device: Perangkat untuk menampilkan hasil (monitor, printer, speaker).
• Storage: Menyimpan data secara permanen (HDD, SSD).

B. Hubungan Arsitektur Von Neumann dengan Model IPO

Pemetaan IPO ke Arsitektur Von Neumann:
• INPUT → Input Device (keyboard, mouse, scanner) → Data masuk ke sistem
• PROSES → CPU (ALU, CU, Register) + Memory (RAM) → Data diolah
• OUTPUT → Output Device (monitor, printer, speaker) → Hasil keluar sistem
📌 Contoh Aliran Data:
1. Input: User mengetik angka 10 dan 5 di keyboard
2. Proses: CPU menghitung 10 × 5 = 50
3. Output: Monitor menampilkan hasil "50"

D. IPO dalam Pseudocode Program

Pseudocode adalah cara menuliskan logika program dengan bahasa yang mudah dipahami, mirip instruksi sehari-hari. Berikut adalah pseudocode program ATM sederhana.

// DEKLARASI VARIABEL saldo = 1000000 ← Data awal (simpanan) pinUser = 123456 ← Data PIN yang benar // ===== INPUT ===== TAMPILKAN "Masukkan kartu ATM" TAMPILKAN "Masukkan PIN" MASUKKAN pinInput ← Data dari user // ===== PROSES ===== IF pinInput == pinUser THEN TAMPILKAN "Masukkan jumlah tarikan" MASUKKAN jumlahTarik ← Data dari user IF jumlahTarik <= saldo THEN saldo = saldo - jumlahTarik ← Proses pengurangan // ===== OUTPUT ===== TAMPILKAN "Uang keluar: " + jumlahTarik TAMPILKAN "Saldo tersisa: " + saldo TAMPILKAN "Cetak struk" ELSE TAMPILKAN "Saldo tidak mencukupi" ENDIF ELSE TAMPILKAN "PIN salah" ENDIF
📌 Identifikasi IPO dalam Pseudocode:
• INPUT: pinInput, jumlahTarik (data yang dimasukkan user)
• PROSES: IF pinInput == pinUser THEN, saldo = saldo - jumlahTarik (verifikasi dan pengurangan)
• OUTPUT: TAMPILKAN "Uang keluar: ", "Saldo tersisa: ", "Cetak struk" (hasil yang ditampilkan)

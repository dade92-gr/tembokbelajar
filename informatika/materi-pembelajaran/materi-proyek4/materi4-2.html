A. Sejarah Arsitektur Von Neumann

Arsitektur Von Neumann adalah desain arsitektur komputer yang pertama kali diperkenalkan oleh matematikawan dan fisikawan John von Neumann pada tahun 1945. Konsep ini menjadi dasar dari hampir semua komputer modern hingga saat ini.

John von Neumann (1903-1957): Matematikawan dan fisikawan Hungaria-Amerika yang pada tahun 1945 menulis laporan "First Draft of a Report on the EDVAC" yang memperkenalkan konsep komputer dengan program tersimpan.
Perkembangan Komputer:
1945: John von Neumann menulis konsep → 1950-an: Komputer pertama berbasis Von Neumann → 1960-70an: Perkembangan mikroprosesor → 1980-90an: Komputer pribadi (PC) → 2000-sekarang: Smartphone, tablet, semua berbasis Von Neumann

B. Konsep Dasar Arsitektur Von Neumann

Konsep utama arsitektur Von Neumann adalah stored-program concept (konsep program tersimpan), di mana instruksi program dan data disimpan dalam memori yang sama.

Prinsip Utama:
• Program dan data disimpan dalam memori yang sama
• Instruksi dieksekusi secara berurutan (kecuali ada lompatan)
• Memori bersifat linear dan dialamati berdasarkan alamat

C. Komponen Utama Arsitektur Von Neumann

Klik pada setiap komponen untuk memperjelas tampilan:

CPU
Central Processing Unit - Otak komputer yang menjalankan instruksi
MEMORI
Menyimpan data dan program (RAM/ROM)
INPUT DEVICE
Menerima data dari luar (keyboard, mouse, scanner)
OUTPUT DEVICE
Menampilkan/mengeluarkan hasil (monitor, printer, speaker)

D. CPU (Central Processing Unit)

CPU terdiri dari tiga bagian utama. Klik pada setiap bagian untuk memperjelas tampilan:

ALU (Arithmetic Logic Unit)
Bertugas melakukan operasi aritmatika (+, -, ×, ÷) dan operasi logika (AND, OR, NOT, perbandingan >, <, =).
CU (Control Unit)
Mengambil instruksi dari memori, menerjemahkan, mengontrol aliran data, dan mengatur timing/sinkronisasi.
Register
Memori sementara dalam CPU: Program Counter (PC), Instruction Register (IR), Memory Address Register (MAR), Memory Data Register (MDR), Accumulator (ACC).

E. Siklus Instruksi (Instruction Cycle)

Klik pada setiap tahap untuk memperjelas tampilan:

1
FETCH
Ambil instruksi dari memori
2
DECODE
Terjemahkan instruksi
3
EXECUTE
Jalankan instruksi
4
STORE
Simpan hasil

F. Hierarki Memori

📝 REGISTER - Paling cepat, kapasitas kecil (dalam CPU)
⚡ CACHE - Cepat, kapasitas sedang (L1, L2, L3 Cache)
💾 RAM - Sedang, kapasitas besar (Random Access Memory)
💿 STORAGE - Lambat, kapasitas sangat besar (HDD/SSD)

G. Hubungan Arsitektur Von Neumann dengan Model IPO

📌 Pemetaan IPO ke Arsitektur Von Neumann:
• INPUT → Input Device (keyboard, mouse, scanner) + Memori
• PROCESS → CPU (ALU, CU, Register) + Memori
• OUTPUT → Output Device (monitor, printer, speaker) + Memori
Contoh Aliran Data (Menghitung Luas Persegi Panjang):
1. Input Device: User mengetik panjang=10, lebar=5
2. Memori: Angka 10 dan 5 disimpan
3. CPU (Fetch): Mengambil instruksi perkalian
4. CPU (Decode): Memahami instruksi perkalian
5. CPU (Execute): ALU menghitung 10 × 5 = 50
6. Memori: Hasil 50 disimpan
7. Output Device: Monitor menampilkan angka 50

H. Kelemahan dan Perkembangan

Von Neumann Bottleneck:
Kecepatan transfer data antara CPU dan memori menjadi penghambat karena instruksi dan data dalam bus yang sama.
✅ Solusi Modern:
• Arsitektur Harvard - Memisahkan bus instruksi dan data
• Cache memory - Memori cepat di dalam CPU
• Pipeline - Eksekusi instruksi paralel

I. Rangkuman

  • Arsitektur Von Neumann adalah desain dasar komputer modern yang diperkenalkan John von Neumann pada 1945.
  • Konsep utama: program dan data disimpan dalam memori yang sama (stored-program concept).
  • 4 komponen utama: CPU, Memori, Input Device, Output Device.
  • CPU terdiri dari: ALU (operasi aritmatika/logika), CU (kontrol), dan Register (memori sementara).
  • Siklus instruksi: Fetch → Decode → Execute → Store.
  • Memori berjenjang: Register → Cache → RAM → Storage.
  • Hubungan dengan IPO: Input device (input), CPU & memori (proses), output device (output).
  • Kelemahan: Von Neumann bottleneck (lambat transfer CPU-memori).
Ingat! Arsitektur Von Neumann adalah fondasi dari semua komputer modern. Memahami konsep ini akan membantumu memahami cara kerja komputer!

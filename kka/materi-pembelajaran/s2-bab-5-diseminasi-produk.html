Implementasi Aplikasi Web dan Pengujian

Semester Genap - Kelas X (Proyek Tahap 4) 8 JP

A. Pendahuluan

Setelah melewati tahap perencanaan (identifikasi masalah), perumusan solusi, dan perancangan (database dan UI/UX), kini saatnya kita memasuki tahap yang paling dinantikan: membangun aplikasi. Tahap ini disebut implementasi atau pengembangan, di mana rancangan yang sudah dibuat diubah menjadi aplikasi yang benar-benar berfungsi.

Bayangkan kamu telah membuat denah rumah yang detail, memilih material bangunan, dan merancang interiornya. Sekarang saatnya membangun rumah tersebut—meletakkan batu bata, memasang kusen, menyusun rangka atap, hingga finishing akhir. Tahap ini adalah proses mewujudkan semua rencana menjadi kenyataan.

B. Apa Itu Implementasi Aplikasi Web?

Implementasi adalah proses mengubah rancangan (desain, database, algoritma) menjadi kode program yang dapat dijalankan oleh komputer. Implementasi meliputi: membangun struktur folder, membuat database, menulis kode backend dan frontend, menghubungkan komponen, serta melakukan pengujian dan perbaikan.

D. Implementasi Database

-- Membuat database CREATE DATABASE perpustakaan_digital; USE perpustakaan_digital; -- Membuat tabel buku CREATE TABLE buku ( id_buku INT PRIMARY KEY AUTO_INCREMENT, judul VARCHAR(200) NOT NULL, penulis VARCHAR(100) NOT NULL, penerbit VARCHAR(100), tahun_terbit YEAR, stok INT DEFAULT 1 ); -- Membuat tabel peminjaman CREATE TABLE peminjaman ( id_peminjaman INT PRIMARY KEY AUTO_INCREMENT, id_buku INT, id_anggota INT, tanggal_pinjam DATE NOT NULL, tanggal_kembali DATE, status ENUM('dipinjam', 'dikembalikan') DEFAULT 'dipinjam', FOREIGN KEY (id_buku) REFERENCES buku(id_buku) );

E. Implementasi Koneksi Database

<?php // config/database.php $host = 'localhost'; $dbname = 'perpustakaan_digital'; $username = 'root'; $password = ''; try { $pdo = new PDO("mysql:host=$host;dbname=$dbname", $username, $password); $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION); } catch(PDOException $e) { die("Koneksi database gagal: " . $e->getMessage()); } ?>

F. Implementasi Model (BukuModel)

<?php class BukuModel { private $db; public function __construct($pdo) { $this->db = $pdo; } public function getAll() { $query = "SELECT * FROM buku ORDER BY id_buku DESC"; $stmt = $this->db->prepare($query); $stmt->execute(); return $stmt->fetchAll(PDO::FETCH_ASSOC); } public function getById($id) { $query = "SELECT * FROM buku WHERE id_buku = :id"; $stmt = $this->db->prepare($query); $stmt->execute(['id' => $id]); return $stmt->fetch(PDO::FETCH_ASSOC); } public function create($data) { $query = "INSERT INTO buku (judul, penulis, penerbit, tahun_terbit, stok) VALUES (:judul, :penulis, :penerbit, :tahun_terbit, :stok)"; $stmt = $this->db->prepare($query); return $stmt->execute($data); } } ?>

H. Jenis-jenis Pengujian

Unit Testing
Menguji komponen terkecil (fungsi, method) secara terpisah
Integration Testing
Menguji interaksi antar komponen
Functional Testing
Menguji fitur dari sudut pandang pengguna
User Acceptance Testing
Menguji dengan pengguna nyata
Security Testing
Menguji keamanan aplikasi (SQL injection, XSS)
Performance Testing
Menguji kecepatan dan beban aplikasi

L. Etika dalam Implementasi

Prinsip Etika Implementasi:
• Keamanan Data: Jangan simpan password plain text, gunakan password_hash()
• Validasi Input: Selalu validasi input untuk cegah SQL injection dan XSS
• Error Handling: Jangan tampilkan detail error teknis ke pengguna
• Hak Cipta: Gunakan library dan aset dengan lisensi sesuai
• Dokumentasi: Dokumentasikan kode agar mudah dipahami orang lain
// ❌ JANGAN: Menyimpan password plain text // $query = "INSERT INTO users VALUES ('$username', '$password')"; // ✅ LAKUKAN: Hash password sebelum disimpan $hashed_password = password_hash($password, PASSWORD_DEFAULT); // ❌ JANGAN: Langsung menggunakan input pengguna di query // $query = "SELECT * FROM users WHERE username = '$username'"; // ✅ LAKUKAN: Gunakan prepared statement $stmt = $pdo->prepare("SELECT * FROM users WHERE username = :username"); $stmt->execute(['username' => $username]);

M. Rangkuman

  • Implementasi adalah proses mengubah rancangan menjadi kode program yang berfungsi.
  • Struktur folder yang rapi memudahkan pengembangan dan pemeliharaan.
  • Model berinteraksi dengan database, Controller memproses logika, View menampilkan tampilan.
  • Pengujian (testing) memastikan aplikasi berfungsi sesuai harapan (Unit, Integration, Functional, UAT, Security).
  • Kasus uji (test case) mendokumentasikan skenario pengujian dan hasil yang diharapkan.
  • Debugging adalah proses mencari dan memperbaiki error dalam kode.
  • Deployment adalah proses meluncurkan aplikasi ke server produksi.
  • Keamanan harus menjadi perhatian utama dalam implementasi.
Ingat! Implementasi adalah tahap di mana ide dan rancanganmu menjadi kenyataan. Pastikan untuk selalu menguji setiap fitur yang kamu buat, karena aplikasi yang baik adalah aplikasi yang berfungsi dengan baik dan aman digunakan!

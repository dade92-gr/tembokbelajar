Analisis Data dan Basis Data

Semester Ganjil - Kelas X 4 JP TP 4.1

A. Apa Itu Data?

Sebelum memahami basis data, kita perlu memahami apa itu data. Data adalah kumpulan fakta, catatan, atau informasi mentah yang belum diolah. Dalam kehidupan sehari-hari, kita selalu berhadapan dengan data, misalnya: daftar nama teman sekelas, catatan pengeluaran bulanan, daftar belanjaan, nilai ujian, dan jadwal pelajaran.

Perbedaan Data dan Informasi:
Data (mentah): 100, 85, 90, 75, 95 | Rp 20.000, Rp 35.000, Rp 15.000
Informasi (setelah diolah): Rata-rata nilai kelas: 89 | Total pengeluaran hari ini: Rp 70.000
Data sendiri belum memiliki makna yang signifikan. Namun, ketika data diolah, ia dapat menjadi informasi yang bermanfaat untuk pengambilan keputusan.

B. Apa Itu Basis Data (Database)?

Basis data (database) adalah kumpulan data yang terstruktur dan tersimpan secara sistematis, sehingga mudah diakses, dikelola, dan diperbarui.

Bayangkan lemari arsip. Jika kamu menyimpan semua dokumen secara acak, kamu akan kesulitan menemukan dokumen yang kamu butuhkan. Namun, jika dokumen disusun rapi dalam folder-folder yang diberi label, kamu bisa menemukannya dengan cepat. Basis data adalah sistem yang melakukan hal itu—mengorganisir data agar mudah ditemukan dan dikelola.

📌 Contoh basis data dalam kehidupan sehari-hari:
• Buku telepon: berisi data nama dan nomor telepon
• Daftar hadir: berisi data nama, tanggal, dan status kehadiran
• Katalog perpustakaan: berisi data judul buku, pengarang, tahun terbit
• Aplikasi dompet digital: menyimpan data saldo, riwayat transaksi, dan profil pengguna

C. Komponen Utama Basis Data

Klik pada setiap komponen untuk memperjelas tampilan (penjelasan tetap terlihat):

📁 Database
Kumpulan data yang terstruktur. Analogi: seluruh lemari arsip. Contoh: Database sekolah.
📊 Tabel
Kumpulan data yang terorganisir dalam baris dan kolom. Analogi: satu laci lemari. Contoh: Tabel Siswa.
🏷️ Field/Kolom
Atribut atau kategori data yang disimpan. Analogi: label pada folder. Contoh: NIS, Nama, Kelas.
📄 Record/Baris
Satu kesatuan data lengkap. Analogi: satu lembar isi folder. Contoh: Data Andi: 001, Andi, X RPL.

Setiap baris = satu record (data lengkap satu siswa)

D. Primary Key (Kunci Utama)

Primary key adalah identitas unik untuk setiap record dalam sebuah tabel. Primary key harus memenuhi syarat: unik (tidak boleh ada dua record dengan nilai yang sama), tidak boleh kosong, dan tidak berubah.

Mengapa primary key penting?
Bayangkan kamu memiliki 5 orang teman yang bernama "Budi" di kelasmu. Jika seseorang bertanya, "Budi mana yang nilai matematikanya 100?", kamu akan bingung. Namun, jika setiap Budi memiliki nomor induk yang berbeda, kamu bisa membedakannya.
📌 Contoh primary key:
• NIS (Nomor Induk Siswa) untuk data siswa
• NIK (Nomor Induk Kependudukan) untuk data penduduk
• ID Transaksi untuk data pembelian
• Nomor polisi untuk data kendaraan

E. Relasi Antar Tabel

Dalam basis data yang baik, data tidak disimpan dalam satu tabel besar, tetapi dipecah menjadi beberapa tabel yang saling berhubungan. Hubungan antar tabel ini disebut relasi.

Jenis-jenis Relasi:

One to One : Satu siswa memiliki satu kartu pelajar
One to Many : Satu guru mengajar banyak mata pelajaran
Many to Many : Banyak siswa mengambil banyak mata pelajaran

Contoh One to Many:
TABEL SISWA ←→ TABEL NILAI
Satu siswa dapat memiliki banyak nilai (One to Many)
Keterangan:
• PK = Primary Key (kunci utama)
• FK = Foreign Key (kunci asing) yang menghubungkan ke tabel lain

F. SQL (Structured Query Language)

SQL adalah bahasa yang digunakan untuk berkomunikasi dengan basis data. Dengan SQL, kita dapat membuat database, membuat tabel, memasukkan data, membaca data, mengubah data, dan menghapus data.

SELECT
Membaca/menampilkan data
SELECT * FROM siswa; (Menampilkan semua data siswa)
INSERT
Memasukkan data baru
INSERT INTO siswa VALUES ('001', 'Andi', 'X RPL');
UPDATE
Mengubah data yang sudah ada
UPDATE siswa SET kelas='X TKJ' WHERE nis='001';
DELETE
Menghapus data
DELETE FROM siswa WHERE nis='001';
-- Contoh Penggunaan SELECT dengan Kondisi
SELECT * FROM siswa; -- Menampilkan semua data siswa
SELECT * FROM siswa WHERE kelas = 'X RPL'; -- Menampilkan siswa kelas X RPL
SELECT nama, nilai FROM siswa WHERE nilai > 85; -- Menampilkan nilai di atas 85
SELECT * FROM siswa ORDER BY nama ASC; -- Menampilkan diurutkan berdasarkan nama

G. Pengolahan Data untuk Pengambilan Keputusan

Data yang telah dikumpulkan dan disimpan dalam basis data perlu diolah agar menghasilkan informasi yang bermakna. Pengolahan data membantu kita menjawab pertanyaan-pertanyaan penting dan mengambil keputusan yang tepat.

📊 Jenis Pengolahan Data:
• COUNT - Menghitung jumlah data (Berapa jumlah siswa di kelas X?)
• SUM - Menjumlahkan nilai (Berapa total pengeluaran bulan ini?)
• AVG - Menghitung rata-rata (Berapa rata-rata nilai ujian matematika?)
• MAX - Mencari nilai tertinggi (Siapa yang mendapatkan nilai tertinggi?)
• MIN - Mencari nilai terendah (Berapa nilai terendah dari ujian?)
• GROUP BY - Mengelompokkan data (Berapa rata-rata nilai per kelas?)
Contoh Kasus:
Seorang guru ingin mengetahui performa siswanya. Dari data nilai yang tersimpan, ia dapat:
1. Menghitung rata-rata kelas: Apakah nilai rata-rata di atas KKM?
2. Mencari nilai tertinggi: Siapa yang perlu diapresiasi?
3. Mencari nilai terendah: Siapa yang perlu bimbingan khusus?
4. Mengelompokkan per mata pelajaran: Mata pelajaran apa yang paling sulit?

I. Mengapa Analisis Data Penting?

Di era digital saat ini, data sering disebut sebagai "minyak baru" (new oil) karena nilainya yang sangat besar. Kemampuan mengolah dan menganalisis data menjadi keterampilan yang sangat berharga di berbagai bidang.

💼
Bisnis
Mengetahui produk terlaris, memprediksi tren pasar
📚
Pendidikan
Mengevaluasi efektivitas pembelajaran
🏥
Kesehatan
Menganalisis penyebaran penyakit
🏛️
Pemerintahan
Mengelola anggaran, merencanakan pembangunan
💰
Keuangan
Mengelola pengeluaran, merencanakan investasi
Contoh Nyata:
Sebuah toko online menganalisis data pembelian pelanggan. Mereka menemukan pola bahwa:
• Produk A paling banyak dibeli pada akhir pekan
• Pelanggan yang membeli produk B biasanya juga membeli produk C
• Diskon 10% meningkatkan penjualan hingga 30%

Dengan temuan ini, toko online dapat menjadwalkan promosi, membuat paket bundling, dan mengoptimalkan strategi diskon.
